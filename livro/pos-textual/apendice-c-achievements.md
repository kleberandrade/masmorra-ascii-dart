# Apêndice C: Achievements do Aventureiro {.unnumbered}

Que tal tornar sua jornada ainda mais épica? Cada seção do livro traz uma série de conquistas que você pode desbloquear conforme avança. Marque o checkbox ao lado assim que completar o desafio. Alguns achievements são bem diretos, outros exigem um toque extra de criatividade. A diversão está em conseguir todos antes de cravar a melhor pontuação possível na masmorra final.

Prepare-se: há alguns achievements secretos escondidos pela floresta. Se você os encontrar, você merece um prêmio especial (spoiler: é a satisfação intelectual).

## Primeiros Passos

Capítulos 1-7: A jornada começa aqui. Você está aprendendo os fundamentos de Dart e ganhando confiança com cada linha de código.

[ ] **Hello, World!** - Compilou e rodou seu primeiro programa Dart no terminal

[ ] **Que Comece o Jogo!** - Criou a estrutura básica do jogo com input e output funcionando

[ ] **Guerreiro Iniciante** - Definiu a classe Jogador com propriedades básicas (nome, vida, ataque)

[ ] **Conhecedor de Tipos** - Usou corretamente pelo menos 5 tipos primitivos diferentes de Dart (int, double, String, bool, List)

[ ] **Organizador de Código** - Criou sua primeira classe e instanciou um objeto com sucesso

[ ] **Mestre das Variáveis** - Declarou corretamente variáveis nulas e não-nulas usando null safety

[ ] **Desafio Extra: Personagem Único** - Criou uma classe Jogador com 3 atributos customizados além dos básicos

## Orientação a Objetos

Capítulos 8-14: O código ganha profundidade. Herança, polimorfismo e abstrações tornam seu jogo muito mais elegante.

[ ] **Construtor Primordial** - Implementou um construtor nomeado em uma classe

[ ] **Getter Dos Deuses** - Usou seu primeiro getter para encapsular uma propriedade privada

[ ] **Herança Domada** - Criou uma classe que estende outra (ex: Guerreiro extends Jogador)

[ ] **Polimorfismo em Ação** - Override de um método da classe pai na classe filha

[ ] **Classes Abstratas Decifradas** - Criou uma classe abstrata e implementou suas funções obrigatórias

[ ] **Inimigo Forjado** - Criou pelo menos 3 tipos de inimigos diferentes com comportamentos distintos

[ ] **Mixin do Poder** - Usou um mixin para adicionar funcionalidade a uma classe

[ ] **Desafio Extra: Hierarquia Completa** - Criou uma árvore de herança com 4+ classes relacionadas

## A Masmorra Ganha Vida

Capítulos 15-21: A masmorra deixa de ser teoria e vira um mundo real. Salas, itens, monstros e sistema de combate entram em cena.

[ ] **Cartógrafo da Masmorra** - Criou um sistema de salas conectadas com no mínimo 10 salas

[ ] **Gerador de Mundos** - Implementou a geração procedural de masmorras (o mundo não é mais estático)

[ ] **Primeira Espada** - Criou o sistema de itens com pelo menos uma arma funcionando

[ ] **Poção Curativa** - Implementou um item que cura dano (poção ou magia de cura)

[ ] **Combate Realizado** - Fez seu primeiro combate entre jogador e inimigo

[ ] **Vitória Épica** - Derrotou um inimigo no combate (vida do inimigo chegou a 0)

[ ] **Sobrevivente** - Tomou dano, se recuperou com uma poção e continuou vivo

[ ] **Desafio Extra: Boss de Andar** - Criou um inimigo especial com vida maior e ataque mais forte que aparece a cada 5 andares

## Economia e Combate

Capítulos 22-27: Moedas de ouro, loot, balanceamento e tática entram na equação. Seu jogo fica viciante.

[ ] **Primeira Moeda de Ouro** - Ganhou sua primeira moeda derrotando um inimigo

[ ] **Caçador de Tesouro** - Coletou ao menos 100 moedas de ouro em uma sessão

[ ] **Loot Épico** - Um inimigo dropou um item raro (chance menor de 25%)

[ ] **Comerciante Experiente** - Comprou um item em uma loja ou vendeu algo

[ ] **Armadura Forjada** - Equipou uma peça de armadura que reduz dano recebido

[ ] **Estratégia Vencedora** - Usou tática no combate (escolheu ataque vs defesa inteligentemente)

[ ] **Derrotado e Revivido** - Morreu mas recarregou um save anterior e venceu a masmorra

[ ] **Desafio Extra: Fortuna de um Rei** - Reuniu 1000+ moedas de ouro sem gastar nada

## Código Profissional

Capítulos 28-32: Testes, tratamento de erro, I/O de arquivos e code smell desaparecem. Seu jogo é mantível.

[ ] **Testador Devotado** - Escreveu seu primeiro teste unitário usando package:test

[ ] **Cobertura Completa** - Escreveu testes para 3+ métodos diferentes

[ ] **Exceção Tratada** - Implementou try/catch para lidar com um erro de forma elegante

[ ] **Save and Load** - Salvou um progresso em arquivo e carregou de volta com sucesso

[ ] **JSON Dominado** - Serializou uma classe complexa para JSON e fez o reverse

[ ] **Debug Eficiente** - Usou print statements ou debugger para encontrar e corrigir um bug

[ ] **Desafio Extra: Teste de Integração** - Escreveu um teste que valida combate completo do início ao fim

## Mestre dos Padrões

Capítulos 33-36: Strategy, Factory, Observer, Command, State. Você agora pensa em padrões. Seu código respira elegância.

[ ] **Factory Fundamental** - Implementou uma Factory para criar inimigos

[ ] **Strategy Sutil** - Diferentes inimigos usam estratégias de IA diferentes

[ ] **Observer Atento** - Usou Stream para notificar múltiplos listeners de um evento

[ ] **Command Armazenado** - Implementou Command pattern para ações do jogo (com execute e desfazer)

[ ] **Estado Máquina** - Usou State pattern para estado de combate ou comportamento de inimigo

[ ] **Sealed Class Power** - Usou sealed class e pattern matching no seu código

[ ] **Desafio Extra: Todos os Padrões** - Integrou todos os 6 padrões principais em um único jogo funcionando

## Achievements Secretos

Desafios especiais que requerem um pouco de criatividade e pensamento lateral. Se você encontrar um, você merece bragging rights.

[ ] **A Verdade está Aqui** - Encontrou uma mensagem de easter egg escondida no código

[ ] **Programador Filozófico** - Refletiu sobre sua própria arquitetura e refatorou por vontade própria

[ ] **Mentorado** - Ajudou alguém a entender um conceito do livro ou resolveu uma dúvida deles

[ ] **Open Source Spirit** - Compartilhou seu código em repositório público (GitHub, GitLab, etc)

[ ] **Extensão Criativa** - Adicionou uma feature completamente original não mencionada no livro

[ ] **Performance Optimizado** - Identificou um gargalo de performance e otimizou

[ ] **Documentação Impecável** - Documentou suas classes com docstrings explicativas

[ ] **Hackathon Pessoal** - Completou o livro inteiro em menos de 30 dias

[ ] **Desafio Ultra: MUD Multiplayer** - Seguiu o Apêndice B e implementou versão em rede do seu jogo

[ ] **Void Spirit** - Entendeu completamente o conceito de `null safety` sem pensar mais uma vez
